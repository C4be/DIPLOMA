\introsection{Актуальность работы}

Бизнес-цели требуют быстрого отклика от аналитиков. Аналитики обращаются к разработчикам, знающим SQL язык для того чтобы получить определенные формы отчетности (Data Marts). Этот процесс медленный, и прослойка между аналитиками, которые общаются на естественном языке и разработчиками, которые общаются с реляционными базами данных, становится рудиментом.

Эта проблема, приводит к нас к созданию NLIDB интерфейсов, для замещения этой прослойке. Раньше были обычные методы, позволяющие получить запрос из человеческой речи путем строго закодированных правил, без осознания контекста. На сегодняшний день, в эпоху LLM моделей и активного развития ИИ и агентов на основе этих моделей. Мы имеем возможность создать интеллектуальную систему для общения с БД.

Тем самым актуальность данной работы обусловлена следующими факторами: экспоненциальный рост данных, высокий порог входа в аналитику, развитие LLM моделей, существующие на момент написания отчета ограничения в сторонних решениях.
