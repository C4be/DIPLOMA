\section{Цель и задачи работы}
\begin{frame}{Цель и задачи работы}
    \textbf{Цель:} Разработка интеллектуального чат-агента для работы с реляционными БД с целью автоматизации процессов формирования отчетности и визуализации данных на основе запросов на естественном языке.
    
    \vspace{0.5cm}
    
    \textbf{Задачи:}
    \begin{itemize}
        \item \textbf{Изучить} современные подходы и программные решения в области \texttt{Text-to-SQL} преобразований и выявить ограничения текущих методов взаимодействия с БД.
        \item \textbf{Спроектировать} архитектуру интеллектуального модуля, включая механизмы обработки естественного языка, логику генерации \texttt{SQL}-кода и компоненты динамической визуализации.
        \item \textbf{Разработать} алгоритмы извлечения контекста на основе технологии \texttt{RAG} для повышения точности формирования запросов к конкретной схеме базы данных.
        \item \textbf{Реализовать} прототип системы с использованием стека технологий \texttt{Python (FastAPI/Streamlit)}, \texttt{PostgreSQL} и интеграцией \texttt{LLM}.
        \item \textbf{Выполнить интеграцию} разработанных компонентов и провести отладку взаимодействия модуля с базой данных.
        \item \textbf{Провести экспериментальную оценку} работоспособности системы, оценив точность генерации запросов и удобство представления аналитических данных. 
    \end{itemize}
\end{frame}


\section{Актуальность}
\begin{frame}{Актуальность: Рост объемов данных и барьер входа в аналитику}
    В настоящий момент мы наблюдаем глобальный рост объемов данных, который, по прогнозам IDC, достигнет 175 зеттабайт. При этом, согласно рейтингу DB-Engines, основным стандартом хранения остаются реляционные базы данных.

    Основная проблема: бизнес-пользователи не знают SQL. По данным Gartner, только четверть сотрудников способны самостоятельно извлекать данные. Это создает узкое горлышко в бизнес-процессах и перегружает разработчиков.
\end{frame}

\begin{frame}{Эволюция методов доступа к данным: от SQL к NLQ (Natural Language Query)}
    Решением является внедрение интерфейсов на естественном языке. 
    Если классические методы генерации SQL работали нестабильно, то современные Большие Языковые Модели показывают высокую точность на таких научных бенчмарках, как Spider и BIRD. 
    
    Используя технологии RAG, можно создать агента, который не просто переводит текст в код, но и понимает структуру базы данных.

    \begin{block}{Цель}
        Цель моей работы — создать систему, которая автоматизирует этот процесс и визуализирует результат.
    \end{block}
\end{frame}
